# 🧼 Bening Laundry - Laundry Hotel SMKN 1 Ciamis
> **Dokumentasi Patch Update — 21 Juni 2026**

Aplikasi web manajemen operasional laundry untuk **Laundry Hotel SMKN 1 Ciamis**. Repositori ini telah diperbarui dengan patch terbaru yang berfokus pada efisiensi alur kerja petugas, akurasi laporan keuangan, kalkulasi kasir real-time, serta standarisasi dokumen cetak (Nota & PDF Laporan).

---

## 🚀 Ringkasan Perubahan pada Patch (21 Juni 2026)

Patch ini mencakup perbaikan bug dan penambahan fitur pada 4 modul utama:

### 1. 💰 Modul Point of Sale (POS) & Nota Digital
* **Input Diskon Dinamis**: Kasir kini dapat memasukkan nominal diskon (potongan harga langsung dalam rupiah) langsung dari halaman pembuatan pesanan POS. Total tagihan akan otomatis terpotong secara dinamis.
* **Nota Fleksibel (Dynamic Receipt)**:
  * Baris **Subtotal** dan **Diskon** hanya akan muncul pada nota jika transaksi memiliki potongan harga (Diskon > Rp 0).
  * Jika tidak ada diskon (Diskon = Rp 0), baris **Subtotal** otomatis disembunyikan untuk menjaga tampilan nota tetap ringkas dan bersih.
* **Integrasi Database**: Data diskon disimpan dengan aman pada tabel `transaksi` (`discount` field).

### 🧺 2. Modul Antrean Petugas Piket (Washing & History)
* **Urutan FIFO (First-In, First-Out)**: Antrean cucian masuk pada portal washing diurutkan berdasarkan waktu pembuatan dari yang terlama ke terbaru. Ini memastikan cucian pelanggan yang masuk lebih awal akan dilayani terlebih dahulu.
* **Desain Ulang Antrean (Tabel Modern)**: Tampilan antrean washing yang sebelumnya berbentuk *grid cards* kini diubah menjadi bentuk **tabel modern** yang ringkas. Mempermudah petugas piket melihat detail antrean, durasi menunggu, dan penggunaan bahan sabun/pewangi.
* **Toast Notification Dinamis**: Notifikasi sukses setelah menyelesaikan cucian kini menampilkan nama pelanggan secara dinamis (Contoh: `"Tugas washing untuk pelanggan Budi berhasil diselesaikan!"`).
* **Riwayat Tugas Padat (Compact History)**: Layout tabel riwayat tugas petugas piket diperkecil ukuran padding dan font-nya (`text-xs`) agar dapat menampilkan informasi yang lebih padat dalam satu layar tanpa perlu banyak scrolling.

### 📊 3. Modul Admin: Laporan Keuangan, Excel & PDF
* **Akurasi Persentase Laba-Rugi**: Memperbaiki perhitungan naik-turun pemasukan dan pengeluaran dibandingkan bulan lalu pada `LaporanController.php`. Persentase sekarang dihitung secara matematis menggunakan nilai riil, bukan lagi sekadar selisih nominal mentah yang diberi simbol persen.
* **Kategori Pengeluaran Dinamis**: Kategori pada form input pengeluaran kini bersifat dinamis, mengambil gabungan opsi default dengan kategori unik yang pernah diinput sebelumnya di database. Status pengeluaran yang tidak perlu juga telah dibersihkan.
* **Penyelarasan Filter Ekspor**:
  * Fitur ekspor Excel dan PDF diselaraskan sepenuhnya dengan filter aktif di halaman laporan (`bulanan`, `tahunan`, dan `custom date range`).
  * File Excel menggunakan pustaka Maatwebsite yang dikonfigurasi dengan auto-size kolom agar data tidak terpotong.
  * Dokumen PDF laporan transaksi didesain ulang dengan format formal: Kop Surat resmi **BENING LAUNDRY SMKN 1 CIAMIS**, tabel bergaris bersih, serta kolom tanda tangan penanggung jawab di bagian kanan bawah.

### 💳 4. Modul Pembayaran Kasir
* **Pembersihan UI**: Menghapus statistik card statis pada bagian atas riwayat transaksi pembayaran agar halaman fokus pada pencarian data pembayaran.
* **Kalkulator Uang Kembalian Real-time**: Mengintegrasikan widget kalkulator interaktif berbasis **Alpine.js** pada form pembayaran. Saat kasir memilih transaksi dan mengetikkan jumlah uang yang dibayarkan pelanggan:
  * Uang kembalian (atau kekurangan bayar) akan terhitung otomatis di layar.
  * Status pembayaran akan otomatis terpilih secara cerdas: **Lunas** (jika pembayaran $\ge$ tagihan) atau **Cicilan** (jika pembayaran < tagihan). Kasir tidak perlu memilih status secara manual lagi.
* **Notifikasi Pelunasan Tunggakan**: Menambahkan pesan notifikasi khusus apabila kasir melunasi transaksi pelanggan yang sebelumnya berstatus `belum_bayar` menjadi `lunas`.

## 🛠️ Cara Menjalankan Aplikasi Secara Lokal

Jika Anda ingin menjalankan atau menguji aplikasi ini dari awal pada mesin lokal Anda:

1. **Persiapan Dependensi PHP & JS**
   ```bash
   composer install
   npm install
   ```

2. **Pengaturan Environment**
   Salin file `.env.example` ke `.env` dan sesuaikan kredensial database Anda (MySQL/MariaDB):
   ```bash
   cp .env.example .env
   php artisan key:generate
   ```

3. **Migrasi Database & Seeder**
   Jalankan migrasi untuk membuat tabel baru atau mengatur ulang database beserta data awal (seeder):
   ```bash
   php artisan migrate:fresh --seed
   ```

4. **Kompilasi Aset Frontend & Jalankan Server Lokal**
   Buka dua jendela terminal terpisah:
   * **Terminal 1 (Vite Dev Server)**:
     ```bash
     npm run dev
     ```
   * **Terminal 2 (Laravel App Server)**:
     ```bash
     php artisan serve
     ```

*Catatan: Anda juga bisa menggunakan file bantu script `setup.bat` yang ada pada root folder untuk memudahkan setup awal otomatis.*
